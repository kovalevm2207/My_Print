~~~~~~~~~~~~
| Правила: |
~~~~~~~~~~~~

'\'                                                                     Используется для переноса длинной строки с командой

## Допустимые символы для меток:

'_', '$', '#', '@',
'.', '?', буквы и цифры

## Префиксы инструкций:

REP
LOCK
REPE/REPZ
REPNE/REPNZ
XACQUIRE/XRELEASE
BND/NOBND

## Префиксы размера адреса и операнда
byte, word, dword
A16, A32, A64
far                                                                     Для косвенных дальних вызовов или переходов

## Псевдоинструкции

DB, DW, DD, DQ, DT, DO, DY и DZ                                         Инициализация данных соответствующего типа

RESB, RESW, RESD, RESQ, REST, RESO, RESY и RESZ                         Объявление неинициализированных данных

INCBIN (сколько пропустить байт) (сколько максимум считать)             Включение внешних двоичных файлов, без аргументов считает весь файл

EQU                                                                     Определение констант

TIMES (сколько повторить) (что повторить)                               Повторение инструкции или данных;
(буфер: база данных 'hello, world'
        times 64-$+buffer db ' ')

0h `<N>` 0x                                                              Шестнадцатеричная
0d `<N>` 0t                                                              Десятеричная
0q `<N>` 0o                                                              Восьмеричная
0b `<N>` 0y                                                              Двоичная


Символы Юникода, указанные с помощью \u`or`, \Uпреобразуются в UTF-8. Например, следующие строки эквивалентны:

      db `\u263a` ; UTF-8 смайлик

$                                                                       Возвращает смещение
$$                                                                      Возвращает начало текущего сегмента

SEG                                                                     Возвращает предпочтительный сегмент символа
        mov     ax,seg symbol
        mov     es,ax
        mov     bx,symbol

.label                                                                  Локальная метка, относительно предыдущей нелокальной
label1.label                                                            Чтобы прыгать на конкретную, локальную метку, а не локальную метку этого сегмента кода
..@                                                                     Метка для макросов, не влияет на механизм работы локальных и нелокальных меток

jmp $                                                                   Трюк с бесконечным циклом

jmpabs target                                                           64-bit absolute jump


Вызов функции, написанной на Си, в Nasm

        1) Как передаются аргументы функции:

                Аргумент 1:     rcx
                Аргумент 2:     rdx
                Аргумент 3:     r8
                Аргумент 4:     r9

           Если параметры функции представляют собой числа с плавающей точкой,
        то они передаются через регистры:

                Аргумент 1:     xmm0
                Аргумент 2:     xmm1
                Аргумент 3:     xmm2
                Аргумент 4:     xmm3

        И дублируются в соответствующих регистрах rcx, rdx, r8 и r9.

                Аргумент 5+:    в стеке начиная со смещения rsp+32:

                        Аргумент n:    rsp+32+(n-5)*8 при условии n >= 5

        Параметры всегда представляют собой 8-байтовое значение.

        2) Возвращаемое значение функции в  rax, если число целое,
        xmm0, если число с плавающей точкой.

        3) Регистры, обязательные для сохранения (Nonvolatile-регистры):

                rbp, rbx, rdi, rsi, rsp, r12 - r15
                xmm6 - xmm15, ymm6 - ymm16

           Регистры, не требующие сохранения     (Volatile-регистры):

                rax, rcx, rdx, r8 - r11
                xmm0 - xmm5, ymm0 - ymm5

        4) Используем точку входа main, а не старт.

        5) После метки global объявляем функцию при помощи директивы extern.

        6) Для вызова функции в стеке необходимо выделить пространство,
        так называемый Shadow space в 32 байта в стеке,
        для хранения и обработки первых четырех аргументов.

        7) Вызов функции производится с помощью:

                call FuncName

        8) При линковке необходимо указать файл с определением функции.
